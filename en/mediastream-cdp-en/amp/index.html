CDP (Customer Data Platform) is an optional module of the Mediastream Platform suite that allows companies to quickly and efficiently access their OTT (Over-The-Top, content distribution channels) user data. This module was designed to facilitate the exploration and analysis of detailed information about user behavior, enabling companies to make more informed decisions and optimize their strategies effectively.

üåü The Project

The challenge was to design an intuitive and efficient module for Mediastream Platform users to easily access their user data and create user funnels to have critical information at their fingertips. This involved designing the UX/UI of the module, creating the funnel system, and testing with users and stakeholders to ensure its effectiveness. Additionally, I developed the initial prototype of the module in React as a microfrontend to integrate it into Mediastream Platform.

Objectives

he project objectives were:

  • Design an accessible and efficient module for user data management.
  • Create an intuitive funnel system to facilitate decision-making.
  • Test the effectiveness of the design with users and stakeholders.
  • Develop the module prototype in React for its integration into the platform.

Phase one: The Problem, Context, and Initial Approaches

For the redesign of the CDP module, I started from the existing Customer module, which showed certain inefficiencies. This module was limited to an informative table with links to user profiles within another module.

The old Customer module did not provide a clear overview of the current customer situation or the overall state of the OTT platform. When the need for a new system arose, it became clear that a solution was needed that would allow for the creation of customer funnels and offer faster and more comprehensive visualization of all data features. The goal was to improve users' ability to analyze and make decisions based on a comprehensive view of the information.

During the development of the CDP module, we took advantage of the Platform Design System update process to validate the new design guidelines in this module. I seized this opportunity to restructure the initial module interface, improving both its efficiency and coherence. I developed new components specific to this module and added dedicated screens for each functionality, ensuring that the design remained in line with the rest of Platform.

To better understand the needs of the module, we built the user profile for the CDP module based on interviews with the commercial area, data analysts, and other professionals who would use the module. These interviews helped us understand their daily tasks, challenges, and objectives, revealing a critical need to access large volumes of data quickly and user-friendly. Based on this information, a detailed user profile was created, represented by Alejandro, the data analyst. This profile guided the module design, ensuring that the needs and expectations of the end users were effectively addressed.

During the interviews, we also discovered a secondary user: in more manual companies and OTTs, we found administrators with little technical knowledge who were responsible for manually updating user data and managing subscriptions and payments. This user was accustomed to how the module already worked, so we took this profile into account to maintain the familiar experience.

Structure and Functionality

Based on the requirements research, we determined that the CDP module had to offer three key actions focused on optimizing data management and decision-making:

  • Access user data: Users should be able to navigate and access detailed data about their OTT users' behavior and preferences. This functionality had to be designed to be intuitive and easy to use, allowing users to quickly find the information they need.
  • Create user funnels: With this option, users could create custom funnels to analyze user behavior at different stages. Funnels would help identify patterns and optimization opportunities, facilitating informed decision-making. This was one of the biggest challenges of the project, as structuring and designing the interaction for funnel creation required a balance between functionality and usability. Additionally, visually handling the large amount of data efficiently was crucial to ensure a smooth and effective user experience.
  • Export data: Users can export data to share with other team members or for further analysis. This functionality is designed to be quick and efficient, ensuring that data can be easily shared. Additionally, we had to retain "legacy" functionalities that were widely used by current users, such as being able to download a list of all deleted customers.

Funnel creation design

For the design of funnel creation in the CDP module, we considered two options: a horizontal funnel and a Sankey diagram. Both options had their merits, but ultimately, the user preferred the Sankey diagrams for being more visually appealing and faster to understand. These diagrams allow for a clear and comprehensible representation of data loss between each filter, which was an important concern for users.

Initially, we considered including both types of graphs, the horizontal funnel and the Sankey diagram, allowing users to toggle between them according to their preferences. This flexibility sought to offer greater customization in data visualization. However, during user testing, we discovered that this duality was confusing and overloaded the user experience. Test participants found it difficult to switch between the two types of graphs and preferred a simpler and more direct solution. This feedback led us to focus solely on the Sankey diagrams, as they provided a clearer and more visually appealing representation of data loss between each filter, facilitating a quick and efficient understanding of the information flow.

We conceived the funnel as a system in which the user starts with the general data (registered users) and adds filters of their preference, visualizing the results of each filter intuitively. The final data of the funnel is displayed on the general screen of the module, providing an overview of the filtered information. Additionally, we included the option to download the final data as a CSV file for further analysis, which responded to a specific need of users to delve deeper into the analysis outside the platform. This functionality was designed to facilitate the decision-making process, allowing users to identify patterns and optimization opportunities efficiently.

To optimize funnel creation in the CDP module, we added the option to apply multiple filters simultaneously, allowing users to build funnels more efficiently and accurately. This functionality was designed for users, like Alejandro, to combine multiple criteria and visualize how they interact in real-time, without needing to apply each filter individually and wait for the results. This improvement not only speeds up the data analysis process but also allows users to experiment with different filter combinations quickly and easily, facilitating the identification of patterns and optimization opportunities. The implementation of this feature was based on feedback received during user testing, highlighting the importance of a flexible and efficient tool for informed decision-making.

User Information

n the original module, there was a user detail screen with detailed and editable information. During the interviews, we determined the need to maintain this screen and its functionalities, but with an optimized design. Additionally, we created an overview screen for the user to offer a quicker and more visual view of the customer and their behavior.

We retained all secondary user functions on the customer screen, optimizing the design and information architecture of the screen. For the overview, we prioritized showing potential insights of the customer profile and behavior with graphs and bullets to facilitate quick reading. This allows users to gain an immediate understanding of patterns and trends without needing to delve into each individual detail. This design decision was based on needs expressed during interviews, highlighting the importance of an interface that is both detailed and accessible.
