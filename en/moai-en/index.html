AI-based chat assistant designed to automate tasks and enhance the user experience on the Mediastream platform.

MoAI (Mediastream AI) is an AI-based chat assistant developed to enhance the user experience on the Mediastream platform by automating and simplifying tasks. This module uses artificial intelligence to perform various functions efficiently, allowing users to focus on more strategic activities.

üåü El Proyecto

MoAI started as an innovation project during the Mediastream hackathon in Chile in February 2024. Over three intense days, we created a functional prototype/proof of concept of a bot that could perform simple tasks within the Mediastream platform. These tasks included cutting a live stream, creating ad markers in media, generating metadata for media, and answering general questions about Mediastream.

Objectives

The project's objectives were:

  • Develop a functional AI assistant capable of automating routine tasks within the Mediastream platform.
  • Design an intuitive user experience for seamless interaction between the assistant and the platform.
  • Present a more polished version of MoAI at the NAB Show to gather feedback and gauge interest.
  • Refine the prototype into a more modular and scalable solution for large-scale implementation.

Initial Phase and Hackathon Experience

During the hackathon, I was responsible for the entire design and flow of the assistant. The focus was on creating an intuitive and user-friendly interface that could handle various tasks efficiently. This phase aimed to create a functional proof of concept that could demonstrate the potential of the AI assistant within the Mediastream platform. The initial prototype was a success, showcasing MoAI's potential to optimize operations within the platform.

The user

The typical MoAI user is a media operator or content producer responsible for managing and optimizing media streaming on the Mediastream platform. These users need to perform repetitive and technical tasks, such as cutting live streams, generating metadata, and creating ad markers. They seek a tool that automates routine processes, improves operational efficiency, and allows them to focus on more strategic and creative activities within the platform.

Key aspects of this phase included:

  • Main Objective: Create an AI assistant capable of executing simple and basic tasks within the Mediastream platform.
  • Basic Functionalities: The prototype allowed actions such as cutting a live stream in Live Editor, creating ad markers in media, and answering general questions about the platform.
  • Interface: The prototype interface was quite simple, based on text commands users entered to interact with the assistant.
  • Conversation Flow: The conversation flow was linear, with a basic structure guiding users through the different available commands without advanced options or personalization.

Improvements for the NAB Show

After the hackathon, we worked on a more polished version of MoAI for its presentation at the NAB Show. This phase involved several key improvements to optimize functionality and user experience:

  • Improved Interface: The interface was redesigned to be more attractive and functional, incorporating buttons and selection options instead of text commands. This facilitated a more intuitive and visual interaction.

For example, on the home screen I included directly visible options, a message that clarifies the context of the chat, and an option to view the conversation history.

To enhance the user experience, we implemented context recognition in the assistant. This means that if the user was viewing multimedia content, the assistant would recognize the media and provide relevant information automatically upon opening. Depending on the context, only relevant options are displayed.

The design was refined to be visually appealing and professional, aligning with the NAB Show's expectations and highlighting the assistant's advanced capabilities.

For the visual design, it was important that the assistant had its own identity. Based on the Platform design system, I created an iteration focused on the secondary purple color and the use of gradients. This helped the assistant look distinctive, but at the same time maintain a familiar and consistent appearance with the rest of the platform.

  • Paths de Conversaci√≥n: Specific conversation paths were implemented for different tasks, with and without context. This allowed the assistant to guide users more effectively through complex workflows, improving usability.

I determined specific guidelines for chat flows, such as:

    • Request Confirmation: User requests or decisions should be repeated by the assistant for confirmation.
    • Option Container: Selectable options are displayed in a separate container for clarity.
    • Selections as Messages: When users select a media/live from an options interface, these selections become a message in the chat. Similarly, quick actions transform into a chat message.
    • Descriptive Title: A header title describing the current chat.
    • Friendly Tone: The assistant's tone and text style are friendly and animated, using emojis and messages divided into paragraphs for better readability.
  • Media Selection Options: More options were introduced to allow users to select the media or live content they wanted to edit, adapting the component to the number of available results.

We implemented this version of the assistant in Platform to present it functionally at the NAB Show. The presentation at the NAB Show was a success, generating significant interest among attendees and stakeholders. This success reinforced the need to implement the assistant definitively in Platform.

Current Production MVP Development

The MVP version of MoAI in production represents the implementation of the assistant's core, utilizing the complete and necessary technology. The features and improvements in this phase include:

  • Metadata Generation: The primary functionality of the MVP focuses on automatic metadata generation for media, using AI-driven transcriptions. This allows users to obtain detailed and accurate metadata more efficiently.
  • Modularity: The MVP is designed to be modular, facilitating the incorporation of new functionalities and adapting to different user needs. This modularity is key for future updates and improvements.

The current iteration of MoAI consists of a simple and efficient flow. The user presses a button that generates metadata based on their configurations. These configurations can be adjusted from the same button, through a secondary option. If the multimedia content does not have generated transcriptions, the assistant requests generating the transcriptions first.

Options for Generating Metadata:

  • Use Account Details: Uses a general account description, included in the general options, about the type of content they usually produce to generate the metadata. This option can be enabled or disabled for each metadata generation. This description is visible from the same modal.

  • Prompt: Specific text about the multimedia content that the assistant uses to generate the metadata.
  • Tone: Selection of the tone for generating the metadata. Available options are:
  • Neutral
  • Informative
  • Formal
  • Relaxed

Currently, MoAI can generate metadata such as the title, description, categories, and tags of media.
